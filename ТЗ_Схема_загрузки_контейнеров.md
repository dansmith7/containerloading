# Техническое задание: Сайт-схема загрузки контейнеров
**Версия:** 1.0  
**Дата:** 30.06.2026  
**Статус:** Прототип утверждён, готов к реализации

---

## 1. Назначение системы

Веб-приложение для автоматического распределения груза (дисплеи, панели, платформы, профили) по морским контейнерам 40HQ. Пользователь вводит состав отгрузки — система рассчитывает оптимальное количество контейнеров, строит визуальную схему загрузки (аналог китайских Excel-схем) и выводит сводную таблицу.

## 3. Справочник ёмкости контейнеров (40HQ)

Данные проверены по реальным отгрузкам:

### 3.1 Чистая загрузка (один тип)

| Товар | Размер | Шт./контейнер 40HQ | Раскладка |
|---|---|---|---|
| Дисплей / Панель / Платформа | 43" | ~160 | L:10×8, R:10×8 |
| Профиль / Дисплей | 55" | 252 | L:14×9, R:14×9 |
| Дисплей / Панель / Платформа | 65" | 160 | L:10×8, R:10×8 |
| Дисплей / Панель / Платформа | 75" | 142 | L:10×7 (70) + R:12×6 (72) |
| Дисплей / Панель / Платформа | 86" | 124 | L:5×11 + R:6×? |

### 3.2 Смешанная загрузка (МИКС 65" + 86")

| Позиция | Кол-во в одном 40HQ |
|---|---|
| 65" (стеллаж сверху) | 80 шт. |
| 86" (нижний ряд) | 62 шт. |
| Итого | 142 ед. |

> **Правило применения:** система автоматически выбирает смешанную загрузку, если это уменьшает суммарное количество контейнеров. Иначе — чистая загрузка.

### 3.3 Расположение в контейнере (двойной стеллаж)

```
┌──────────┬──────────┐
│ Левая    │ Правая   │
│ колонна  │ колонна  │
├──────────┼──────────┤
│ 10 шт.   │ 12 шт.  │ ← 75", ряд 1
│ (ряд 1)  │ (ряд 1)  │
├──────────┼──────────┤
│ ...      │ ...      │
└──────────┴──────────┘
     40HQ CONTAINER
```

## 4. Типы груза и категории

```
display   — Дисплей (DHR): 43", 55", 65", 75", 86"
panel     — Панель (SL-A311D2): 65", 75", 86"
platform  — Платформа (SL-A311D2): 65", 75", 86"
profile   — Профиль: 43", 55", 65"
spare     — Запасная часть (едет с основным грузом)
accessory — Аксессуары (шурупы, крепёж) — не занимают отдельный контейнер
```

Для расчёта ёмкости: **display = panel = platform** по одному размеру (одинаковые физические габариты).

## 5. Экраны и пользовательский сценарий

### Экран 1: Ввод груза (главный)

**Левая панель — форма ввода:**
- Блок «Тип контейнера» → выбор 40HQ / 20GP (расширение)
- Кнопка «+ Добавить позицию» → добавляет строку с полями:
  - Тип товара (выпадающий список)
  - Размер (дюймы, зависит от типа)
  - Количество (число)
- Кнопка «Рассчитать» → запускает алгоритм

**Правая область — пустой стейт:**
- Инструкция: «Добавьте позиции слева и нажмите Рассчитать»

### Экран 2: Результат

**Верхняя строка — сводка:**
```
[12] Контейнеров 40HQ  |  [1 875] Единиц груза  |  [94%] Заполненность  |  [3] Смешанных конт.
```

**Вкладка «Схема контейнеров»:**
- Карточки контейнеров в сетке (3-4 в ряд на 1920px)
- Каждая карточка содержит:
  - Заголовок: `40HQ-1`, тип (МИКС / Чистый), % заполнения
  - SVG-визуализация: вертикальный контейнер, два столбца, цветные блоки по рядам
  - Прогресс-бар заполненности
  - Легенда: какой груз, сколько штук

**Вкладка «Сводная таблица»:**

| № Конт. | Тип | Состав груза | Кол-во | Ёмкость | Заполнено |
|---|---|---|---|---|---|
| 40HQ-1 | Чистый | Платформа 75" | 142 | 142 | 100% |
| 40HQ-2 | МИКС | Панель 65" | 80 | 142 | 100% |
|  |  | Панель 86" | 62 |  |  |

**Вкладка «Справочник ёмкости»:**

- Встроенная документация

## 6. Алгоритм распределения (авто-режим)

```
ВХОД: список { тип, размер, количество }

1. Разделить на "основной груз" и "запчасти/аксессуары"

2. Проверка оптимизации МИКС (65" + 86"):
   a. Если есть оба размера (65" + 86") не-профильного товара:
      - pureNeeded = ceil(qty65/160) + ceil(qty86/124)
      - mixedFull = min(floor(qty65/80), floor(qty86/62))
      - mixedNeeded = mixedFull + ceil(остаток65/160) + ceil(остаток86/124)
   b. Если mixedNeeded < pureNeeded → применяем МИКС, сообщаем об экономии
   c. Иначе → чистая загрузка

3. Заполнение МИКС-контейнеров:
   - Берём по 80 × 65" и 62 × 86" за раз
   - До тех пор, пока оба остатка >= минимальной доли

4. Остатки каждого размера → чистые контейнеры:
   - ceil(qty / capacity) контейнеров на размер

5. Нумерация: 40HQ-1, 40HQ-2, ...

ВЫХОД: список контейнеров с содержимым
```

## 7. Визуальная схема контейнера (SVG)

Референс: третий скриншот (китайская схема загрузки).

```
┌─────── 40HQ ────────┐
│ ┌───────┬──────────┐ │
│ │ L col │  R col   │ │  ← ряд 1 (верхний)
│ │ 10шт  │  12шт    │ │
│ ├───────┼──────────┤ │
│ │ 10шт  │  12шт    │ │  ← ряд 2
│ ├───────┼──────────┤ │
│ │ ...   │  ...     │ │
│ └───────┴──────────┘ │
│      [ДВЕРЬ]         │
└──────────────────────┘
```

**Цветовая кодировка:**
- 43" → зелёный `#43a047`
- 55" → бирюзовый `#00897b`
- 65" → синий `#1e88e5`
- 75" → фиолетовый `#8e24aa`
- 86" → красный `#e53935`

**Метки на рядах** (как в референсе): `75双层10台` → `75" ×10`

## 8. Ручной режим

Переключатель «Ручной режим» позволяет:
- Самостоятельно задать состав каждого контейнера
- Перетаскивать позиции между контейнерами (drag & drop)
- Добавить контейнер вручную
- Система подсвечивает перегруз (красный) и недогруз (<60%, оранжевый)
